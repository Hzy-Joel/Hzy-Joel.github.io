### Glide源码

- Glide的初始化过程：

1. 在使用Glide.with方法时会传入一个context，该方法返回一个GlideRequests对象，如下：
```
	Glide.with(this)//在Activity中调用
```
在Glide4.4之后使用的是GlideApp代替Glide，但实际原理不变

2. Glide.with方法中getRetriever方法返回一个RequestManagerRetriever对象：
```
	@NonNullpublic static RequestManager with(@NonNull Activity activity) {  
        return getRetriever(activity).get(activity);
	}
```
3. Glide类中的getRetriever方法：
```
	@NonNull
	private static RequestManagerRetriever getRetriever(@Nullable Context context) {
  //上下文可能由于其他原因而为空（即，用户传入null），但实际上，它仅会由于Fragment生命周期错误而发生调用时传入的是空对象
    Preconditions.checkNotNull(
        context,
        "You cannot start a load on a not yet attached View or a Fragment where getActivity() "
            + "returns null (which usually occurs when getActivity() is called before the Fragment "
            + "is attached or after the Fragment is destroyed).")；
		return Glide.get(context).getRequestManagerRetriever();
	}
```
在该方法中通过Glide.get(context)方法初始化Glide。

4. Glide中的get方法：
```
	@NonNull
	public static Glide get(@NonNull Context context) {
    if (glide == null) {
    //获取模块 --- 这里通过注解@GlideModule在编译时用户自定义的模块生成自己工程下的GeneratedAppGlideModuleImpl类，因此在反射时类加载器会先加载本地jar包路径的生成的GeneratedAppGlideModuleImpl类，而默认初始化中反射使用Glide的默认GeneratedAppGlideModuleImpl类
      GeneratedAppGlideModule annotationGeneratedModule =
          getAnnotationGeneratedGlideModules(context.getApplicationContext());
      synchronized (Glide.class) {
        if (glide == null) {
          checkAndInitializeGlide(context, annotationGeneratedModule);
        }
      }
    }c

    return glide;
  }
```
通过DLC单例模式获取，在第一次调用时进入checkAndInitializeGlide方法初始化Glide，其中会加载用户自定义的模块。

5. Glide中的checkAndInitializeGlide方法防止Glide的get方法多个线程调用导致的多次初始化，真正的初始化在initializeGlide中：
```
	private static void initializeGlide(
      @NonNull Context context,
      @NonNull GlideBuilder builder,
      @Nullable GeneratedAppGlideModule annotationGeneratedModule) {
    Context applicationContext = context.getApplicationContext();
    //从AndroidManifest中解析配置文件，在新版本的已经不需要配置，新版本的自定义在get方法中通过反射获取，详细的在后面会提到。
    ...
   // debug 输出日志
    ...
    RequestManagerRetriever.RequestManagerFactory factory =
        annotationGeneratedModule != null
            ? annotationGeneratedModule.getRequestManagerFactory()
            : null;
            
    //设置加载配置
    builder.setRequestManagerFactory(factory);
    for (com.bumptech.glide.module.GlideModule module : manifestModules) {
      module.applyOptions(applicationContext, builder);
    }
    if (annotationGeneratedModule != null) {
      annotationGeneratedModule.applyOptions(applicationContext, builder);
    }
    
    //创建 glide实例
    Glide glide = builder.build(applicationContext);
    //注册自定义配置类Module的加载配置，注册对应编码解码器等。
    ...
    //注册状态回调，四大组件都注册了该接口用于接收管理内存变化、窗口变化等
    applicationContext.registerComponentCallbacks(glide)；
    Glide.glide = glide;
  }
```
在上面的代码中主要是加载了用户自定义的模块，并使用GlideBuilder设置了该配置生成的一个RequestManagerFactory，然后使用GlideBuilder的builder方法创建了一个glide对象。

6. GlideBuilder的builder:
```
  @NonNull
  Glide build(@NonNull Context context) {
  //初始化一系列缓存池
    if (sourceExecutor == null) {
      sourceExecutor = GlideExecutor.newSourceExecutor();
    }

    if (diskCacheExecutor == null) {
      diskCacheExecutor = GlideExecutor.newDiskCacheExecutor();
    }

    if (animationExecutor == null) {
      animationExecutor = GlideExecutor.newAnimationExecutor();
    }

    if (memorySizeCalculator == null) {
      memorySizeCalculator = new MemorySizeCalculator.Builder(context).build();
    }

    if (connectivityMonitorFactory == null) {
      connectivityMonitorFactory = new DefaultConnectivityMonitorFactory();
    }

    if (bitmapPool == null) {
      int size = memorySizeCalculator.getBitmapPoolSize();
      if (size > 0) {
        bitmapPool = new LruBitmapPool(size);
      } else {
        bitmapPool = new BitmapPoolAdapter();
      }
    }

    if (arrayPool == null) {
      arrayPool = new LruArrayPool(memorySizeCalculator.getArrayPoolSizeInBytes());
    }

    if (memoryCache == null) {
      memoryCache = new LruResourceCache(memorySizeCalculator.getMemoryCacheSize());
    }

    if (diskCacheFactory == null) {
      diskCacheFactory = new InternalCacheDiskCacheFactory(context);
    }

    if (engine == null) {
    //创建加载引擎类
      engine =
          new Engine(
              memoryCache,
              diskCacheFactory,
              diskCacheExecutor,
              sourceExecutor,
              GlideExecutor.newUnlimitedSourceExecutor(),
              animationExecutor,
              isActiveResourceRetentionAllowed);
    }

    if (defaultRequestListeners == null) {
      defaultRequestListeners = Collections.emptyList();
    } else {
      defaultRequestListeners = Collections.unmodifiableList(defaultRequestListeners);
    }

    RequestManagerRetriever requestManagerRetriever =
        new RequestManagerRetriever(requestManagerFactory);

    return new Glide(
        context,
        engine,
        memoryCache,
        bitmapPool,
        arrayPool,
        requestManagerRetriever,
        connectivityMonitorFactory,
        logLevel,
        defaultRequestOptionsFactory,
        defaultTransitionOptions,
        defaultRequestListeners,
        isLoggingRequestOriginsEnabled
        isImageDecoderEnabledForBitmaps);
  }
```
在代码中可以看出缓存池的初始化实在GlideBuilder中初始化的，最后调用了Glide的构造函数，构造函数中传入了各类缓存池和在一开始Glide.with方法中getRetriever方法返回的RequestManagerRetriever对象就是在这里创建的。

8. RequestManagerRetriever的get方法:

```
	@NonNull
	public RequestManager get(@NonNull FragmentActivity activity) {
		if (Util.isOnBackgroundThread()) {
		//不是在主线程
		  return get(activity.getApplicationContext());
		} else {
		  assertNotDestroyed(activity);
		  FragmentManager fm = activity.getSupportFragmentManager();
		  return supportFragmentGet(activity, fm, /*parentHint=*/ null, isActivityVisible(activity));
		}
	}
```

该方法返回通过supportFragmentGet方法生成的RequestManager。

9. RequestManagerRetriever的supportFragmentGet方法：

``` 
	@NonNull
	private RequestManager supportFragmentGet(
      @NonNull Context context,
      @NonNull FragmentManager fm,
      @Nullable Fragment parentHint,
      boolean isParentVisible) {
		SupportRequestManagerFragment current =
			getSupportRequestManagerFragment(fm, parentHint, isParentVisible);
		RequestManager requestManager = current.getRequestManager();
		if (requestManager == null) {
			// TODO(b/27524013): Factor out this Glide.get() call.
			Glide glide = Glide.get(context);
			requestManager =
			  factory.build(
				glide, current.getGlideLifecycle(), current.getRequestManagerTreeNode(), context);
			current.setRequestManager(requestManager);
		return requestManager;
	}
```

上面的代码中可以看出requestManager是由factory的build方法中创建的，但在这之前，需要看下其中build传入的参数是如何来的，主要是如何实现Glide和Activity的生命周期关联。

10. Glide和Activity的生命周期关联，SupportRequestManagerFragment的生成：
在上面的代码中SupportRequestManagerFragment是由getSupportRequestManagerFragment方法生成的：

```
	private SupportRequestManagerFragment getSupportRequestManagerFragment(
      @NonNull final FragmentManager fm, @Nullable Fragment parentHint, boolean isParentVisible) {
      //通过Tag获得Fragment
		SupportRequestManagerFragment current =
			(SupportRequestManagerFragment) fm.findFragmentByTag(FRAGMENT_TAG);
		if (current == null) {
		  current = pendingSupportRequestManagerFragments.get(fm);
		  if (current == null) {
			//获取为空的话就创建一个
			current = new SupportRequestManagerFragment();
			current.setParentFragmentHint(parentHint);
			if (isParentVisible) {
			   //调用onStart回调
			  current.getGlideLifecycle().onStart();
			}
			pendingSupportRequestManagerFragments.put(fm, current);
			//添加到对应的context的FragmentManger中，附加tag
			fm.beginTransaction().add(current, FRAGMENT_TAG).commitAllowingStateLoss();
			handler.obtainMessage(ID_REMOVE_SUPPORT_FRAGMENT_MANAGER, fm).sendToTarget();
		  }
		}
		return current;
	}
```

Glide通过向Activity中添加了一个固定Tag的Fragment，并在这个Fragment中实现了一个Lifecycle接口的子类用于通知外界生命周期的变化。因此每个Activity中如果调用了with都会有一个空的Fragment,互不干扰。

11. 回到第9步中使用RequestManagerFactory的build生成一个RequestManager对象（首次调用的时候）：RequestManagerFactory在初始化中Glide的initializeGlide方法以下代码通过

```
	RequestManagerRetriever.RequestManagerFactory factory =
        annotationGeneratedModule != null
			? annotationGeneratedModule.getRequestManagerFactory(): null;
```

**annotationGeneratedModule.getRequestManagerFactory()该方法返回null，具体实现子类为GeneratedAppGlideModuleImpl，在子类GeneratedAppGlideModuleImpl中的getRequestManagerFactory()返回GeneratedRequestManagerFactory类，因此此处的factory调用的是GeneratedRequestManagerFactory的build方法（如果在没有自定义GlideModule的情况下，默认情况下在Glide的getAnnotationGeneratedGlideModules方法中通过反射获取到到这个GeneratedAppGlideModuleImpl类，其中其他的调用方法是调用用户自定义模块的重写，因此也可以重写具有注解@GlideModule的AppGlideModule类的getRequestManagerFactory方法自定义一个Factory类）**

12. 默认的GeneratedRequestManagerFactory的build方法：

```
	public RequestManager build(@NonNull Glide glide, @NonNull Lifecycle lifecycle,
		@NonNull RequestManagerTreeNode treeNode, @NonNull Context context) {
			return new GlideRequests(glide, lifecycle, treeNode, context);
		}
	}
```

该方法返回一个GlideRequests

13. GlideRequests类继承于RequestManager，且该类是由编译生成的，在编译时注解@GlideExtension和@GlideType会被动对应生成，因此使得Glide.with方法返回的对象所调用的链式方法可以由@GlideExtension和@GlideType这两个注解自定义添加特定的方法处理。由于该类继承自RequestManager，其中的实现都在RequestManager父类中，在该类中储存了请求的配置。

14. 在使用RequestManager类方法时会返回一个带泛型的GlideRequest<T>类，该类继承于RequestBuilder<T>类，该泛型用于转换请求结果的类型转换和类型检查，这个类是创建用于保存请求的内容Request类，直到调用RequestBuilder<T>的into方法，数据才开始加载。

15. RequestBuilder的into方法：

```
  private <Y extends Target<TranscodeType>> Y into(
      @NonNull Y target,
      @Nullable RequestListener<TranscodeType> targetListener,
      BaseRequestOptions<?> options,
      Executor callbackExecutor) {
    Preconditions.checkNotNull(target);
    if (!isModelSet) {
      throw new IllegalArgumentException("You must call #load() before calling #into()");
    }
    
    //创建一个请求体
    Request request = buildRequest(target, targetListener, options, callbackExecutor);

    Request previous = target.getRequest();
    //重复的请求的话就重新开始，并将target返回
    if (request.isEquivalentTo(previous)
        && !isSkipMemoryCacheWithCompletePreviousRequest(options, previous)) {
      // If the request is completed, beginning again will ensure the result is re-delivered,
      // triggering RequestListeners and Targets. If the request is failed, beginning again will
      // restart the request, giving it another chance to complete. If the request is already
      // running, we can let it continue running without interruption.
      if (!Preconditions.checkNotNull(previous).isRunning()) {
        // Use the previous request rather than the new one to allow for optimizations like skipping
        // setting placeholders, tracking and un-tracking Targets, and obtaining View dimensions
        // that are done in the individual Request.
        previous.begin();
      }
      return target;
    }

    //清除请求缓存f
    requestManager.clear(target);
    target.setRequest(request);
    //开始请求
    requestManager.track(target, request);

    return target;
  }
```

最终的请求在判断是重复请求后会调用previous.begin重新执行，不是重复的请求则会调用requestManager.track(target, request)开始加载。

15. requestManager.track(target, request)方法：

```
 synchronized void track(@NonNull Target<?> target, @NonNull Request request) {
    targetTracker.track(target);
    requestTracker.runRequest(request);
  }
```

其中的 targetTracker.track(target);是将这个target对象保存起来，用于在RequestManager类中注册的生命周期回调的通知。
requestTracker.runRequest(request);则是正式开始加载。

16. requestTracker.runRequest(request)方法：

```
 /** Starts tracking the given request. */
  public void runRequest(@NonNull Request request) {
    requests.add(request);
    if (!isPaused) {
      request.begin();
    } else {
      request.clear();
      if (Log.isLoggable(TAG, Log.VERBOSE)) {
        Log.v(TAG, "Paused, delaying request");
      }
      //保存请求以进行下一次排队
      pendingRequests.add(request);
    }
  }
```

这里调用开始加载的方法也是Request类的begin方法，与之前判断重复请求的方式一样。

17. Request类的begin()方法，由于该类是一个接口，需要找到实际实现的类，回到第15步，该类实例化是由RequestBuilder的buildRequest(target, targetListener, options, callbackExecutor);方法生成。

18.RequestBuilder的buildRequest方法：

```
 private Request buildRequest(
      Target<TranscodeType> target,
      @Nullable RequestListener<TranscodeType> targetListener,
      BaseRequestOptions<?> requestOptions,
      Executor callbackExecutor) {
    return buildRequestRecursive(
        /*requestLock=*/ new Object(),
        target,
        targetListener,
        /*parentCoordinator=*/ null,
        transitionOptions,

  public void begin() {
    synchronized (requestLock) {
    //错误检测
      assertNotCallingCallbacks();
      stateVerifier.throwIfRecycled();
      startTime = LogTime.getLogTime();
      if (model == null) {
        if (Util.isValidDimensions(overrideWidth, overrideHeight)) {
          width = overrideWidth;
          height = overrideHeight;
        }
        // Only log at more verbose log levels if the user has set a fallback drawable, because
        // fallback Drawables indicate the user expects null models occasionally.
        int logLevel = getFallbackDrawable() == null ? Log.WARN : Log.DEBUG;
        onLoadFailed(new GlideException("Received null model"), logLevel);
        return;
      }

      if (status == Status.RUNNING) {
        throw new IllegalArgumentException("Cannot restart a running request");
      }

      // If we're restarted after we're complete (usually via something like a notifyDataSetChanged
      // that starts an identical request into the same Target or View), we can simply use the
      // resource and size we retrieved the last time around and skip obtaining a new size, starting
      // a new load etc. This does mean that users who want to restart a load because they expect
      // that the view size has changed will need to explicitly clear the View or Target before
      // starting the new load.
      if (status == Status.COMPLETE) {
        //触发回调
        onResourceReady(resource, DataSource.MEMORY_CACHE);
        return;
      }

      // Restarts for requests that are neither complete nor running can be treated as new requests
      // and can run again from the beginning.

      status = Status.WAITING_FOR_SIZE;
        //判断是否需要重新获取大小，最终都会调用到onSizeReady方法重新启动获取。
      if (Util.isValidDimensions(overrideWidth, overrideHeight)) {
        onSizeReady(overrideWidth, overrideHeight);
      } else {
        target.getSize(this);
      }

      if ((status == Status.RUNNING || status == Status.WAITING_FOR_SIZE)
          && canNotifyStatusChanged()) {
        target.onLoadStarted(getPlaceholderDrawable());
      }
      if (IS_VERBOSE_LOGGABLE) {
        logV("finished run method in " + LogTime.getElapsedMillis(startTime));
      }
    }

      //取得解码后的转换
      Transformation<?> transformation = helper.getTransformation(resourceClass);
     
      currentKey =
          new ResourceCacheKey( // NOPMD AvoidInstantiatingObjectsInLoops
              helper.getArrayPool(),
              sourceId,
              helper.getSignature(),
              helper.getWidth(),
              helper.getHeight(),
              transformation,
              resourceClass,
              helper.getOptions());
              
       //获取到对应的磁盘缓存资源文件
      cacheFile = helper.getDiskCache().get(currentKey);
      if (cacheFile != null) {
        sourceKey = sourceId;
        //获得注册的解码加载器
        modelLoaders = helper.getModelLoaders(cacheFile);
        modelLoaderIndex = 0;
      }
    }

    loadData = null;
    boolean started = false;
    while (!started && hasNextModelLoader()) {
       //遍历解码器
      ModelLoader<File, ?> modelLoader = modelLoaders.get(modelLoaderIndex++);
      //创建一个解码数据类型
      loadData =
          modelLoader.buildLoadData(
              cacheFile, helper.getWidth(), helper.getHeight(), helper.getOptions());
      // helper.hasLoadPath(loadData.fetcher.getDataClass()这句代码说明解码器可以解码该类型
      if (loadData != null && helper.hasLoadPath(loadData.fetcher.getDataClass())) {
        //只有解码成功才会走到这里
        started = true;
        //调用解码器解析数据
        loadData.fetcher.loadData(helper.getPriority(), this);
      }
    }
    return started;
  }
```
可以看到最终调用的是loadData.fetcher.loadData(helper.getPriority(), this);这个方法解码数据，因此要找到该fetcher对应的具体初始化地方，可以看到loadData数据是由modelLoader.buildLoadData方法生成，modelLoader是由对应的类获取到的解码器，回到modelLoader的初始化地方，看下Glide默认注册了那些解码器：

  public boolean startNext() {
    while (modelLoaders == null || !hasNextModelLoader()) {
      sourceIdIndex++;
      if (sourceIdIndex >= cacheKeys.size()) {
        return false;
      }
        
      Key sourceId = cacheKeys.get(sourceIdIndex);
      // PMD.AvoidInstantiatingObjectsInLoops The loop iterates a limited number of times
      // and the actions it performs are much more expensive than a single allocation.
      //生成的图片Key值
      @SuppressWarnings("PMD.AvoidInstantiatingObjectsInLoops")
      Key originalKey = new DataCacheKey(sourceId, helper.getSignature());
      cacheFile = helper.getDiskCache().get(originalKey);
      if (cacheFile != null) {

        modelLoaders = helper.getModelLoaders(cacheFile);
        modelLoaderIndex = 0;
      }
    }

    loadData = null;
    boolean started = false;
